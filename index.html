> My hatred of CVS has meant that I see Subversion as being the most pointless > project ever started ... [their] slogan was 'CVS done right' ... > if you start with that kind of slogan, there is nowhere you can go > ... there is no way to do CVS right. \- Linus Torvalds

Why use version control?

## Version History Those who cannot remember the past...
#### Versions with git * a log of every change * includes metadata about the change * easily compare versions
## Backups
#### Git Backups * restore is a common operation * easy to find a specific version * data is compressed * bad for binary files
## Collaboration
#### Git as a collaboration tool * many changes can be automatically merged * decentralized * everyone can have their own "fork" * every change includes metadata
#### Do everything with the same tool! * version history * backups * collaboration
## How?
## Terminology * repository — a local database of "snapshots" of files * git client — a piece of software which allows you to interact with the repository

Getting Started

Create a new git repository

git init

$ git init first-git-project
Initialized empty Git repository in /home/daniel/first-git-project/.git/

$ ls first-git-project/.git/

config
description

info/
objects/
refs/

Get the status of files in our repository 

git status

$ git status
On branch master

Initial commit

nothing to commit

Adding files to our repository 

git add

$ touch README
$ git status

Untracked files:
        README

$ git add README
$ git status

Changes to be committed:
        new file:   README

Commit files to our repository 

git commit

$ git commit -m "Initial commit."

[master (root-commit) 1cfcc37] Initial commit.
 1 file changed, 0 insertions(+), 0 deletions(-)
 create mode 100644 README

$ git status

On branch master
nothing to commit, working directory clean

See the difference 

git diff

$ vim README
$ git diff

diff --git a/README b/README
index e69de29..51d27c4 100644
--- a/README
+++ b/README
@@ -0,0 +1,5 @@
+
+This is my first git repo
+=========================
+
+Some text here.

$ git status

Changes not staged for commit:
        modified:   README

Add our changes to the repository 


$ git add README
$ git commit -m "Add some text to the README."

[master 5474bdf] Add some text to the README.
1 file changed, 5 insertions(+)

$ git status

On branch master
nothing to commit, working directory clean

Using SSH


$ ssh user@backup.dn.com

$ mkdir /var/git
$ git init --bare /var/git/first-git-project.git
Initialized empty Git repository in /var/git/first-git-project.git/

Adding a remote repository 

git remote

$ git remote add origin git@github.com:dnephin/first-git-project.git

$ git remote add backup user@backup.dn.com:/var/git/first-git-project.git

$ cat .git/config

[remote "origin"]
    url = git@github.com:dnephin/first-git-project.git
    fetch = +refs/heads/*:refs/remotes/origin/*
[remote "backup"]
    url = user@backup.dn.com:/var/git/first-git-project.git
    fetch = +refs/heads/*:refs/remotes/backup/*
